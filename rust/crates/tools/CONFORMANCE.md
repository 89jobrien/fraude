# Conformance

## External Standards

- **JSON Schema Draft 7**: tool input schemas generated by the `macros` crate and exposed
  via `ToolSpec` follow JSON Schema Draft 7 subset (type, description, properties, required).
  Validation constraints (minimum, pattern, enum) are not enforced at this layer.

## Informal Contracts

- **Tool name format**: tool names are lowercase, underscore-separated identifiers
  (e.g. `read_file`, `bash`). Names must be stable across versions as they appear in
  LLM-generated tool calls.
- **`allowed_tools` normalisation**: `normalize_allowed_tools` expands glob patterns
  and resolves aliases before the list is passed to the LLM.

## Known Deviations

- Input schema validation is not performed at call time; malformed tool arguments are
  passed through to the handler and may produce opaque errors.
- `reqwest` HTTP calls inside custom tools bypass the permission system; this is a
  known gap pending a permission hook on outbound HTTP.
